# 架空カフェLP 仕様メモ

HTML/CSS/JavaScriptだけで作る、架空カフェの1ページランディングページです。  
まずは実装方針とファイル構成を整理し、次の工程でHTML/CSS/JavaScriptを実装します。

## 目的

- 架空カフェの雰囲気、メニュー、こだわり、アクセス情報を1ページで伝える
- 予約・問い合わせへの導線を用意する
- スマホでも見やすく、ナビゲーションしやすいLPにする
- 外部APIやフレームワークに依存しない静的サイトとして作る

## 技術方針

- HTML/CSS/JavaScriptのみで実装する
- React、Next.js、ビルドツールは使わない
- 外部APIは使わない
- JavaScriptはスマホメニューの開閉など、必要最小限にする
- 個人情報は掲載しない
- 画像サービスには依存せず、CSSの背景、色、余白、装飾で雰囲気を作る
- スマホ表示を優先し、レスポンシブ対応を必ず行う

## 想定するカフェ設定

- 店名: Komorebi Cafe
- コンセプト: 街の小さな路地にある、木漏れ日と焼き菓子の香りを楽しむカフェ
- トーン: 落ち着き、あたたかさ、清潔感
- 主な訴求: 自家焙煎コーヒー、季節の焼き菓子、ひとりでも過ごしやすい空間

## ページ構成

### 1. Hero

- 店名を大きく表示する
- キャッチコピーを配置する
- CTAボタンを配置する
  - 例: 「メニューを見る」「アクセスを見る」
- ファーストビューでカフェの雰囲気が伝わるように、CSS背景や装飾を使う

### 2. About

- カフェの紹介文を掲載する
- 架空店舗として、個人情報や実在する住所・人物名は使わない
- 「静かな時間」「焼きたて」「自家焙煎」などの価値を短く伝える

### 3. Menu

- ドリンクとフードを掲載する
- 価格は架空のものにする
- カードまたはリスト形式で、スマホでも読みやすく配置する
- 例:
  - ブレンドコーヒー
  - カフェラテ
  - 季節のハーブティー
  - スコーン
  - チーズケーキ
  - クロックムッシュ

### 4. Features

- こだわりを3つ掲載する
- 例:
  - 自家焙煎の豆
  - 季節の焼き菓子
  - ひとりでも過ごしやすい席づくり

### 5. Access

- 営業時間、架空の住所、アクセス案内を掲載する
- 実在住所と誤認されにくい架空情報にする
- 例:
  - 営業時間: 8:00 - 19:00
  - 定休日: 水曜日
  - 住所: 東京都みどり区こもれび町 1-2-3
  - アクセス: こもれび駅 南口から徒歩5分

### 6. Contact

- 予約・問い合わせへの導線を配置する
- 実際のフォーム送信は行わない
- ボタンや案内テキストで「予約する」「問い合わせる」導線を表現する
- メールアドレスや電話番号など、個人情報に見える情報は掲載しない

### 7. Footer

- 店名
- 著作権風の表記
- ページ内リンク

## ナビゲーション方針

- PCではヘッダーに横並びのナビゲーションを表示する
- スマホではハンバーガーメニューにする
- ナビゲーション項目はページ内アンカーにする
  - About
  - Menu
  - Features
  - Access
  - Contact
- スマホメニューはJavaScriptで開閉する
- メニューを開いた状態でリンクを押したら閉じる
- `aria-expanded` などを使い、アクセシビリティにも配慮する

## デザイン方針

- CSSだけでカフェらしい雰囲気を作る
- 余白を広めに取り、読みやすいレイアウトにする
- 色はあたたかみのあるベースに、コーヒーや植物を連想する差し色を使う
- セクションごとに背景色や装飾を少し変え、単調にならないようにする
- ボタン、カード、見出し、ナビゲーションの状態が分かるようにする
- フォーカス状態を見えるようにし、キーボード操作にも配慮する

## レスポンシブ方針

- モバイルファーストでCSSを書く
- スマホでは1カラムを基本にする
- タブレット以上ではMenuやFeaturesを2〜3カラムにする
- ヘッダーは固定または追従しすぎず、画面を圧迫しない設計にする
- テキストやボタンが小さくなりすぎないようにする

## JavaScript方針

JavaScriptで担当する機能は最小限にします。

- ハンバーガーメニューの開閉
- メニュー内リンククリック時のメニュー閉じ処理
- 必要であれば、フッターの年表示などの軽い補助処理

フォーム送信、外部通信、複雑なアニメーションは実装しません。

## ファイル構成案

```text
cafe-landing-page/
├── AGENTS.md
├── README.md
├── index.html
├── styles.css
└── script.js
```

補足: 現在のフォルダには `styles.cdd` がありますが、実装時は一般的な拡張子である `styles.css` を使う方針です。

## 実装時の注意点

- HTMLは `header`, `main`, `section`, `footer` など意味のあるタグを使う
- 各セクションにページ内リンク用の `id` を付ける
- ボタンやリンクの文言は、行き先や目的が分かるものにする
- 架空情報であっても、実在の個人情報に見える内容は避ける
- フォームを置く場合も送信処理は作らず、デモ用であることが分かる形にする
- 実装後にREADMEへ表示確認方法を追記する

## 完了条件

- 1ページLPとして必要なセクションがすべて揃っている
- PCとスマホでレイアウトが崩れない
- スマホでハンバーガーメニューが動作する
- 外部API、React、Next.jsを使っていない
- 個人情報を掲載していない
- Contactは導線のみで、実際の送信処理を行わない

## ローカルでの表示確認方法

このLPは静的なHTML/CSS/JavaScriptだけで動作するため、ビルドや依存関係のインストールは不要です。

1. `cafe-landing-page/index.html` をブラウザで開く
2. 画面幅を狭くして、ハンバーガーメニューが開閉することを確認する
3. ナビゲーションやCTAボタンを押して、各セクションへ移動できることを確認する
4. Contactのデモフォームで「送信せずに確認する」を押し、実際の送信が行われないことを確認する

ローカルサーバーで確認したい場合は、フォルダ内で次のように起動できます。

```bash
python3 -m http.server 8000
```

起動後、ブラウザで `http://localhost:8000` を開きます。
